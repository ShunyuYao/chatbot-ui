## Codebase Patterns
- **DB Migrations**: `npm run db-migrate` won't work without local Docker — update `supabase/types.ts` manually, then apply SQL via Supabase Management API. Use `IF NOT EXISTS` for safety.
- **types.ts format**: Tables sorted alphabetically, each entry has Row/Insert/Update/Relationships; fields within each block also sorted alphabetically
- **Supabase project ref**: `mlfpqlxqwdifklwunnqe` (extracted from NEXT_PUBLIC_SUPABASE_URL)
- **Typecheck**: `node_modules/.bin/tsc --noEmit` — pre-existing warnings in lint are not caused by our changes
- **UI Select**: Radix UI Select is at `@/components/ui/select` — import `Select, SelectTrigger, SelectValue, SelectContent, SelectItem`
- **Model forms**: create-model.tsx and model-item.tsx both use the pattern: state hooks → createState/updateState → renderInputs; new DB fields can be added via the `as TablesInsert/TablesUpdate` cast
- **Custom model routes**: Use `createClient` with service role key to query models table; `getServerProfile()` is for built-in provider routes only
- **Anthropic streaming**: `AnthropicStream` from `ai` library works with `anthropic.messages.create({ stream: true })` responses — but does NOT support thinking blocks
- **Extended thinking**: SDK v0.18.0 lacks thinking types — use `Record<string, any>` for createParams and cast `anthropic.messages.create as Function` to bypass type errors; pass `anthropic-beta` header via RequestOptions second argument
- **Extended thinking streaming**: Use custom ReadableStream with `for await` over the response (AsyncIterable); track block types via `content_block_start` event's `content_block.type` field; thinking deltas use `thinking_delta` type with `delta.thinking` field
- **Thinking + temperature**: Anthropic requires `temperature: 1` when thinking is enabled; max_tokens should be at least `budget_tokens + 4096`
- **ChatSettings**: Does not have `maxTokens` — custom routes should use a hardcoded default (4096)
- **Chat routing**: Custom model routing is in `handleHostedChat` in `components/chat/chat-helpers/index.ts` — `apiEndpoint` ternary on ~line 221
- **Thinking + streaming**: `parseThinkingContent` helper handles both complete `<thinking>...</thinking>` blocks and incomplete blocks mid-stream (no closing tag yet)
- **DB stripping**: `stripThinkingTags` in chat-helpers/index.ts removes thinking tags before persisting — call it before any `createMessages` or `updateMessage`

- **Learnings for future iterations:**
  - SDK v0.18.0 has no `thinking` param or `thinking_delta` event types — use type assertions to bypass
  - Anthropic streaming response is an AsyncIterable — use `for await (const event of response)` for manual handling
  - `content_block_start` events include `event.index` and `event.content_block.type` — use a Map to track which index corresponds to which block type
  - `content_block_stop` events include `event.index` — use it to look up the block type from the Map
  - Anthropic API requires `temperature: 1` when thinking is enabled
