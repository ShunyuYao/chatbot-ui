## Codebase Patterns
- `ChatSettings` component is a self-contained popover button (model name + gear icon) that reads/writes `chatSettings` from `ChatbotUIContext`
- `npm run type-check` requires `tsc` via local binary: use `node_modules/.bin/tsc --noEmit`
- Pre-existing TS errors in playwright tests and image imports are not caused by our changes
- The chat UI layout in `chat-ui.tsx` uses absolute positioning: top-left for scroll buttons, top-right for action buttons
- `ChatUI` `loading` state starts `true`, set to `false` after `fetchData()` completes; use to guard against firing effects on initial load
- ChatSettings fields use camelCase; DB columns use snake_case (e.g., `contextLength → context_length`)
- `updateChat` in `db/chats.ts` accepts `TablesUpdate<"chats">` with all fields optional
- **Radix UI**: `PopoverTrigger` must use `asChild` when its child is a `<Button>` component, otherwise renders `<button>` inside `<button>` causing a DOM nesting violation and React console error
- **dev-browser**: To send a chat message, use `locator("textarea").first().focus()` then `keyboard.press("Enter")` — `getByPlaceholder` times out on this component
- **dev-browser**: `getByRole("button", { name: "..." })` throws strict mode error if multiple matches exist — use `.first()` to disambiguate
