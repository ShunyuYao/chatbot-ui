## Codebase Patterns
- **Share page pattern**: Server components can directly use `createClient` from `@supabase/supabase-js` with service role key for unauthenticated DB access; pair with a client component (share-message-list.tsx) for rendering React-based components like `MessageMarkdown`
- **Middleware bypass**: Add `if (request.nextUrl.pathname.match(/\/share\//)) return NextResponse.next()` before the supabase auth check to allow unauthenticated access to share pages
- `ChatSettings` component is a self-contained popover button (model name + gear icon) that reads/writes `chatSettings` from `ChatbotUIContext`
- `npm run type-check` requires `tsc` via local binary: use `node_modules/.bin/tsc --noEmit`
- Pre-existing TS errors in playwright tests and image imports are not caused by our changes
- The chat UI layout in `chat-ui.tsx` uses absolute positioning: top-left for scroll buttons, top-right for action buttons
- `ChatUI` `loading` state starts `true`, set to `false` after `fetchData()` completes; use to guard against firing effects on initial load
- ChatSettings fields use camelCase; DB columns use snake_case (e.g., `contextLength → context_length`)
- `updateChat` in `db/chats.ts` accepts `TablesUpdate<"chats">` with all fields optional
- **Radix UI**: `PopoverTrigger` must use `asChild` when its child is a `<Button>` component, otherwise renders `<button>` inside `<button>` causing a DOM nesting violation and React console error
- **dev-browser**: To send a chat message, use `locator("textarea").first().focus()` then `keyboard.press("Enter")` — `getByPlaceholder` times out on this component
- **dev-browser**: `getByRole("button", { name: "..." })` throws strict mode error if multiple matches exist — use `.first()` to disambiguate
- **DB Migrations**: `npm run db-migrate` won't work without local Docker — update `supabase/types.ts` manually, then apply SQL via Supabase Management API (see root CLAUDE.md). `.gitignore` has `*.sql` but migration files are allowed via `!supabase/migrations/*.sql`
- **types.ts format**: Tables sorted alphabetically, each entry has Row/Insert/Update/Relationships; `shared_chats` is between `prompts` and `tool_workspaces`
- **Toast notifications**: Use `sonner` (see root CLAUDE.md). ⚠️ Do NOT use `useToast` from `@/components/ui/use-toast`.
- **GitHub CLI auth**: Set `GH_TOKEN` env var (from `.env.local`) — no interactive login needed. Must use a **classic token** (`ghp_`), NOT fine-grained PAT (`github_pat_`); fine-grained PATs lack GraphQL `mergePullRequest` permission and will get HTTP 403 on merge.
- **Auto-merge setup**: Before first use, enable auto-merge on the repo: `gh api repos/{owner}/{repo} -X PATCH -f allow_auto_merge=true`. Then `gh pr merge --auto --squash`. If no branch protection rules exist, the PR merges immediately.
